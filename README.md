# CampLus

**校园学术生活的操作系统。**

---

## 问题

今天一个中国大学生的学术信息散落在至少五个地方：

- QQ 群公告里找作业截止时间
- 微信群聊天记录里翻教师的通知
- 辅导员的消息和闲聊混在一起，重要的被刷掉
- 选课靠打听——这门课老师怎么样？给分什么分布？没人知道
- 好的笔记和复习资料锁在个人电脑里，学期结束就再也没人看到

这些信息**存在**，但**不在该在的地方**。每次想找什么，都是一次考古。

## CampLus 要做什么

把一个学生与"课程、知识、选课、信息"相关的全部需求，**整合到一个平台里**。不是再做一個论坛、一个聊天工具、或一个教务系统。是一个 **让学生自己维护、自己治理的学术信息基础设施**。

---

## 核心系统

| 系统 | 做什么 |
|---|---|
| **课程仪表盘** | 每门课的结构化主页。按周组织的 Timeline（公告、作业、考试、资料）作为信息中枢，学生自主编辑维护，教师可选参与 |
| **个人首页** | 我修的课、本周截止时间、学业进度。不塞信息流，只放你需要知道的 |
| **知识广场** | 笔记和知识库的协作空间。Notion 式的块编辑器 + GitHub 式的 PR 协作流程。课程可以引用笔记，笔记可以引用课程 |
| **选课 × 评课** | 搜索引擎 + 课程数据档案（历年成绩分布、评价聚合）+ 可视化 Schedule Builder。选课不再靠玄学 |
| **组织与社团** | 每个社团一个公开主页，活动管理和公告发布 |

## 核心理念

### 学生自治——不是口号，是战略选择

平台**在没有任何教师参与的情况下完整运转**。课程信息由学生上传、集体编辑、集体审核。教师可以接入并标注"官方确认"，但不是系统运转的前提。

这不仅仅是理想。**一旦系统依赖学校行政配合，就会在审批和对接中被卡死。**

### 实名——信任的基础

一切发言和内容贡献都与一个真实的人绑定。没有匿名。实名是问责的前提，问责是自治的前提。

### 结构化信息，不是信息流

没有全站信息流，没有热度排行，没有推荐算法。内容属于它所属的实体（课程、知识库、组织）。你来找你需要的，不是来消磨时间的。

---

## 不是什么

- ❌ 不是聊天工具——没有群聊和即时对话
- ❌ 不是树洞——没有匿名、没有八卦
- ❌ 不是社交媒体——没有点赞、转发、关注
- ❌ 不是游戏——没有积分、等级、徽章
