<p align="center">

## 为什么需要 CampLus

一个中国大学生的学术信息，今天大概分散在五个以上的地方：QQ 群公告、微信群聊天、Piazza、教务系统、某个同学的电脑。它们之间不互通，无法搜索，学期一结束就沉底。

几个太常见的场景：

- 想知道某门课的作业截止时间 → 打开 QQ 群 → 搜索聊天记录 → 往上翻到三天前的公告 → 发现日期又改了
- 想选课 → 问学长"这门课怎么样" → 回复是"还行" → 你没有更多数据可以参照
- 辅导员发了一份重要通知 → 群消息顶到上面去了 → 往上翻 → 再翻 → 算了不找了
- 期末想找一份同学做的复习笔记 → 群文件翻了两百个 → 没找到 → 可能根本没上传过

**信息存在，但不在它该在的地方。** CampLus 要解决的，就是这个。

---

## CampLus 做什么

一个**让学生自主维护、自主治理的学术信息基础设施**。不需要等学校审批，不需要教师入驻才能运转。五个核心模块：

| 模块 | 解决什么 |
|---|---|
| 📚 **课程仪表盘** | 每门课一个结构化主页。按教学周组织的 Timeline（公告、作业、考试、资料），学生编辑维护，所有变更留痕可追溯 |
| 🏠 **个人首页** | 只看你需要关心的：修读课程、本周截止时间、学业进度。不塞信息流 |
| 📝 **知识广场** | Notion 式块编辑器 + GitHub 式 PR 协作。写笔记、搭知识库，课程和笔记互相引用但互不隶属 |
| 🔍 **选课 × 评课** | 课程搜索 → 数据档案（历年成绩分布、评价聚合）→ Schedule Builder 可视化排课。选课不靠玄学 |
| 🏢 **组织与社团** | 每个社团一个公开主页，发活动、发公告。不用加群才能了解 |

## 几条硬规矩

| 原则 | |
|---|---|
| **实名** | 一切发言和内容贡献绑定真实身份。没有匿名，没有马甲。这是问责和信任的前提 |
| **学生自治** | 系统在没有任何教师参与的情况下完整运转。教师可选择接入，但不是必要前提 |
| **结构化信息** | 没有全站信息流、热度榜、推荐算法。内容归属于实体，搜索是意图驱动的 |
| **权力即责任** | 更大的编辑权限伴随更明确的记录和更可追溯的审计 |

## 不是什么
